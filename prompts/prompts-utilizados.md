# Prompts Utilizados no NotebookLM

## Objetivo

Este documento reúne os principais prompts utilizados durante o estudo de **SQL para Análise de Dados e Business Intelligence** no NotebookLM.

O objetivo foi explorar diferentes formas de interação com a IA para aprofundamento técnico, revisão de conteúdo e criação de material reutilizável para estudos futuros.

---

# Prompt 1 — Explicação Conceitual

## Objetivo

Entender JOINs de forma didática e contextualizada.

### Prompt:

```text
Explique JOIN em SQL como se estivesse ensinando alguém que está migrando do desenvolvimento backend para análise de dados.
```

### Resultado esperado:

* visão conceitual clara;
* comparação com relacionamento entre tabelas;
* exemplos práticos.

---

# Prompt 2 — Comparação Técnica

## Objetivo

Compreender diferenças práticas entre os tipos de JOIN.

### Prompt:

```text
Qual a diferença prática entre INNER JOIN, LEFT JOIN e RIGHT JOIN? Mostre exemplos com tabelas fictícias.
```

### Resultado esperado:

* comparação objetiva;
* visualização prática de cada tipo de retorno;
* entendimento aplicado em cenários reais.

---

# Prompt 3 — Exercícios Práticos

## Objetivo

Praticar SQL com foco em análise de dados.

### Prompt:

```text
Crie exercícios práticos com nível júnior para praticar GROUP BY e HAVING.
```

### Resultado esperado:

* exercícios voltados para BI;
* prática de agregações;
* fixação do conteúdo estudado.

---

# Prompt 4 — Resumo Técnico

## Objetivo

Consolidar os principais comandos SQL em uma revisão rápida.

### Prompt:

```text
Resuma os principais comandos SQL usados em Business Intelligence.
```

### Resultado esperado:

* material resumido para revisão;
* visão geral rápida dos comandos mais importantes.

---

# Prompt 5 — Glossário

## Objetivo

Criar um material de apoio para futuras consultas.

### Prompt:

```text
Monte um glossário com os 20 principais conceitos aprendidos sobre SQL para BI.
```

### Resultado esperado:

* definição dos principais termos;
* referência rápida para revisão futura.

---

# Prompt 6 — Simulação de Entrevista Técnica

## Objetivo

Preparação para entrevistas técnicas na área de dados.

### Prompt:

```text
Quais perguntas técnicas sobre SQL podem ser feitas em entrevistas para análise de dados ou BI?
```

### Resultado esperado:

* preparação para entrevistas;
* revisão prática aplicada ao mercado.

---

# Prompt 7 — Explicação aplicada em negócio

## Objetivo

Conectar SQL ao contexto de negócio e análise de indicadores.

### Prompt:

```text
Explique GROUP BY com exemplos voltados para dashboards de vendas e análise de negócio.
```

### Resultado esperado:

* contexto real de negócio;
* aplicação em relatórios e dashboards;
* conexão entre SQL e BI.

---

# Aprendizados sobre Engenharia de Prompt

Durante o processo foi possível perceber que pequenas mudanças na formulação das perguntas geram respostas muito mais úteis.

## Principais aprendizados:

* prompts específicos geram respostas melhores;
* incluir contexto técnico melhora a profundidade;
* pedir exemplos reais aumenta aplicabilidade prática;
* direcionar para BI gera respostas mais próximas do uso profissional.

---

# Conclusão

A utilização do NotebookLM mostrou que a IA pode ser uma ferramenta poderosa de aprendizagem ativa quando combinada com boas fontes e prompts bem estruturados.

Documentar os prompts utilizados também ajuda na reutilização futura e permite acompanhar a evolução do raciocínio ao longo do estudo.
